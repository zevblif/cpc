## 2 各参数依据

### 2.1 环维度 `m = 256`

- **下界**：`m` 必须为 2 的幂以支持 NTT；过小则 Ring‑SIS 安全性不足。
- **取值**：与 Dilithium‑III 一致，Core‑SVP 成本估计 ~128 比特。
- **上界**：`m = 512` 可达 ~256 比特，但 `σ`、`B`、证明大小均线性增长。当前选 `m = 256` 兼顾安全与紧凑。

### 2.2 模数 `q = 8 380 417`

- **NTT 友好**：`q ≡ 1 (mod 2m) = 1 (mod 512)`，故 `2m` 次单位根存在，可在 `R` 内做完整 `m` 点 NTT。
- **素数**：`q` 为素数，保证 `Z_q` 是域，环 `R` 是 Dedekind 整环。
- **大小**：`q < 2^23`，每个系数可用 23 位 ≈ 3 字节编码，序列化紧凑。
- **来源**：直接复用 Dilithium‑III 的素数，避免重新做 NTT 友好性数论分析。

### 2.3 挑战重量 `τ = 60`

- **挑战空间大小**：`|C| = C(m, τ) · 2^τ ≈ 2^260`，足够抵抗 `~2^128` 次随机预言机查询的分叉攻击（分叉成功概率 `1/|C|` 可忽略）。
- **可逆性**：引理 2 要求 `||Δc^{-1}||∞ ≤ 2τ`；`τ = 60` 给出逆范数界 `120`，提取范数 `4τ√m·B ≈ 2.4×10^9`。
- **来源**：与 Dilithium‑III 一致，复用其 `SampleInBall` 与可逆性证明。

### 2.4 步长界 `β = 45`

- **应用层约束**：路径步长 `||d_i|| ≤ β` 由具体应用决定（如位置证明中相邻采样点的位移上界）。
- **取值**：`β = 45` 是示例值；只要满足 `σ = 12·τ·β` 与 `B = 1.2·σ·√m` 由此导出，且 `ν = 2B + 2τ·β·√m` 落在 Ring‑SIS 困难区间，方案均安全。
- **可调**：`β` 是 `PublicParams` 的可配置项，不同应用可选用不同 `β`。

### 2.5 高斯宽度 `σ = 12·τ·β = 32 400`

- **拒绝采样引理**要求 `σ = τ_ratio · T`，其中 `T = β·√m` 是秘密范数上界，`τ_ratio ≥ 12` 保证 `M = exp(12/τ_ratio + 1/(2τ_ratio²)) ≈ 2.7`。
- 注意：论文中 `τ` 一词在两处出现。`τ = 60` 是挑战重量；拒绝采样引理中的 `τ_ratio = 12` 是 `σ/T` 之比。为避免混淆，代码中 `TAU = 60`（挑战重量），`σ` 直接以数值形式给出。
- **取值**：`σ = 12 · 60 · 45 = 32 400`。

### 2.6 响应截断界 `B = 1.2·σ·√m ≈ 622 080`

- 高斯尾部界：`||z|| ≤ σ·√m` 以压倒性概率；乘以 `1.2` 作为安全余量。
- 验证者据此检查 `||z|| ≤ B`，拒绝超出者。
- **提取范数**：`||d*|| ≤ 2τ·√m · 2B = 4τ·√m·B`，代入得 `4·60·16·622080 ≈ 2.39×10^9`。

### 2.7 路径长度 `L = 1 024`

- 仅影响 Merkle 树高度 `⌈log₂ L⌉ = 10`，不影响承诺大小（仍为 32 B）。
- 证明中 Merkle 路径含 10 个 32 字节兄弟哈希，共 320 字节。
- `L` 可达 `2^32` 而不改变承诺大小，仅增大路径长度（线性于 `log L`）。

### 2.8 拒绝采样上限 `MAX_REJECT_ITERATIONS = 1 000`

- 期望迭代 `M ≈ 2.7`；`1000` 次仍失败的概率为 `(1 - 1/M)^1000 ≈ exp(-370)`，可忽略。
- 工程兜底，防止异常输入（如错误的 `σ`）导致死循环。

## 3 安全性评估

### 3.1 Ring‑SIS 实例

```
m = 256,  q ≈ 2^23,  ν ≈ 2.4×10^9 ≈ 2^31.2
```

### 3.2 Core‑SVP 估计

在 BKZ 块大小 `κ` 下，Core‑SVP 成本 `2^(0.292·κ)`，安全位数 `≈ 0.292·κ`。对上述实例：

- 经典 Core‑SVP：`~128` 比特
- 量子 Core‑SVP（Groner 加速）：`~118` 比特

与 Dilithium‑III 的安全级别一致（NIST Level 3，~128 位经典安全）。

### 3.3 哈希与随机预言机

- `H_MT`：SHA3‑256，抗碰撞 ~128 比特，绑定归约匹配。
- `H`（挑战）：SHAKE‑256 抽取，ROM 下分叉引理适用。
- `q_H`：实际中敌手哈希查询次数受计算能力限制，`q_H ≤ 2^128` 时定理 3 的 `ε²/(q_H+1)` 仍有意义。

## 4 参数一致性检查清单

实现过程中需在测试中验证：

- [ ] `σ == 12 * TAU as f64 * BETA as f64`（编译期断言）
- [ ] `(B as f64) - 1.2 * SIGMA * (M as f64).sqrt() < 1.0`（容许浮点误差）
- [ ] `EXTRACT_NORM_BOUND == 4 * TAU * M * B`
- [ ] `Q % (2 * M as i64) == 1`（NTT 友好性）
- [ ] `Q` 为素数（编译期难以断言，写在单测中）

这些检查列入 [implementation-plan.md](implementation-plan.md) Task 1.1。
